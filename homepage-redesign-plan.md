## 5 个区域详细设计

### 1. Hero 区域
- 大字标题「远方音乐」（约 2rem，粗体）
- 副标题显示曲库统计：`共 N 首曲目 · M 位艺人`（从 `playlistStore.allTracks` 计算）
- 「浏览全部音乐」链接 → `/browse`
- 简洁无冗余装饰

### 2. 精选艺人网格
- 从所有曲目中提取艺人，按曲目数排序取前 8 位
- 复用 `ArtistCard.vue` 组件（已有）
- 响应式 CSS Grid：桌面 4 列 → 平板 3 列 → 手机 2 列
- 点击跳转到艺人详情页 `/artist/:id`

### 3. 绿色音乐可视化背景（核心新增）
- **纯 CSS 实现**，无需外部图片/GIF
- 约 350px 高度，深色半透明背景
- 品牌绿色 `#1DB954` + 发光阴影 `box-shadow`

**三种状态：**

| 播放状态 | 视觉表现 |
|---------|---------|
| 无播放 (`currentTrack === null`) | 居中音乐符号 + 「选择歌曲开始播放」引导文字 |
| 播放中 (`isPlaying === true`) | 多根绿色柱状条跳动（不同高度 + 延迟） |
| 暂停 (`isPlaying === false`) | 柱状条静止，显示「已暂停」 |

**CSS 均衡器动画：**
```css
@keyframes equalizer {
  0%, 100% { height: 15%; }
  20% { height: 60%; }
  40% { height: 30%; }
  60% { height: 80%; }
  80% { height: 45%; }
}
/* 约 20-30 根柱条，每根不同 animation-delay */
```

- **底部始终显示**：当前歌曲名 + 艺人名（有播放时），或引导文字（无播放时）

### 4. 精选曲目（默认歌单）
- 从 `allTracks` 中取前 10 首（或第一次加载时固定选取）
- 展示为可点击列表：序号、歌名、艺人、时长
- 点击某行 → 播放该歌曲
- 「▶ 播放全部」按钮
- 当前播放的行用绿色高亮

**数据来源保障：**
```typescript
const featuredTracks = computed<Track[]>(() => {
  return allTracks.value.slice(0, 10)  // 真实存在的曲目
})
```

### 5. 最近播放
- 从 `userStore.playHistory` 反查曲目信息
- 横向滚动列表，显示歌名 + 艺人
- 最多 10 首
- 无历史记录时隐藏该区域
